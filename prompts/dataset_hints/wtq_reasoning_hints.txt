Additional WTQ reasoning hints:
- For singular min/max questions such as "which country has the least..." or "who has the most...", if multiple rows tie and the question does not ask for all ties, return the first matching answer in table order rather than all tied answers.
- For questions using "first", "last", "top", "bottom", "on the chart", or "on the list", preserve the table/list order unless the question explicitly asks for chronological order, numeric rank order, or valid finishers only. For "finished last", keep both table/list order and Result/Rank status columns in scope; do not rewrite it only as maximum numeric Rank.
- For WTQ questions involving "21st century" and a Year column, consider the benchmark/table convention before enforcing a strict historical boundary; do not automatically exclude year 2000 if the table context treats it as the first relevant 21st-century entry.
- For WTQ CSV code generation, prefer `pd.read_csv(file_path, encoding="utf8", escapechar="\\")` before falling back to skipped bad lines; using `on_bad_lines="skip"` can silently drop relevant rows.
- For count questions grouped by county/category, if the matching category appears in separate non-contiguous table sections, avoid blindly counting every substring match across the whole file. Consider whether the WTQ question refers to a local table section, original grouping, or rows surfaced in table context.
